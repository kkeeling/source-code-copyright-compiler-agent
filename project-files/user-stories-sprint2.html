User Stories: Source Code Copyright Compiler - Sprint 2

US-05: File and Folder Filtering

As a user,
I want to be able to specify folders, file extensions, and specific files to ignore,
So that I can exclude irrelevant or sensitive parts of my project from the copyright document.

Acceptance Criteria:

  • Default ignore lists for common folders and file extensions are implemented
  • Users can add custom folders, extensions, and files to ignore via command-line arguments
  • The tool correctly filters out specified items during project analysis
  • Ignored items are logged for user reference

Tasks:

T-05-01: Implement FilterManager class with default ignore patterns (ID: 5001)

T-05-02: Update CLI to accept custom ignore patterns (ID: 5002)

T-05-03: Integrate FilterManager with Project Analyzer (ID: 5003)

T-05-04: Implement logging for ignored items (ID: 5004)

T-05-05: Write unit tests for FilterManager (ID: 5005)

US-06: Gemini LLM Integration for Code Prioritization

As a user,
I want the tool to use Google's Gemini 1.5 Flash LLM to prioritize my project's code,
So that the most important parts of my codebase are highlighted in the copyright document.

Acceptance Criteria:

  • The tool successfully integrates with the Gemini 1.5 Flash LLM API
  • Project code is sent to the LLM for analysis and prioritization
  • The LLM's response is correctly parsed to extract prioritized file contents
  • Files are ordered in the final document according to the LLM's prioritization

Tasks:

T-06-01: Set up Gemini LLM API integration using Google AI Python SDK (ID: 6001)

T-06-02: Implement code preparation and submission to LLM (ID: 6002)

T-06-03: Develop parsing logic for LLM responses (ID: 6003)

T-06-04: Integrate LLM prioritization with document generation process (ID: 6004)

T-06-05: Implement error handling and retries for API calls (ID: 6005)

US-07: Sensitive Information Obfuscation

As a user,
I want the tool to automatically obfuscate sensitive information in my code,
So that I can safely submit my copyright document without exposing secrets or keys.

Acceptance Criteria:

  • The tool detects common patterns of sensitive information (e.g., API keys, passwords)
  • Detected sensitive information is replaced with placeholder text
  • Obfuscation occurs before code is included in the final document
  • A log of obfuscated items is generated for review

Tasks:

T-07-01: Implement regex patterns for detecting sensitive information (ID: 7001)

T-07-02: Develop obfuscation logic to replace sensitive data (ID: 7002)

T-07-03: Integrate obfuscation process with code processing pipeline (ID: 7003)

T-07-04: Implement logging system for obfuscated items (ID: 7004)

T-07-05: Create unit tests for obfuscation functionality (ID: 7005)

US-08: Enhanced Document Generation

As a user,
I want the tool to generate a document with only the first and last 10 "pages" of each important file,
So that I have a concise yet comprehensive representation of my project for copyright purposes.

Acceptance Criteria:

  • The document includes only the first and last 10 "pages" of code for each file
  • Files are presented in order of importance as determined by the LLM
  • Each file section includes a clear header with the file name
  • The document maintains proper formatting and readability

Tasks:

T-08-01: Modify Code Processor to extract first and last 10 "pages" of code (ID: 8001)

T-08-02: Update Document Generator to create structured content based on LLM prioritization (ID: 8002)

T-08-03: Implement file name header generation for each code section (ID: 8003)

T-08-04: Ensure proper formatting and styling of the MS Word document (ID: 8004)

T-08-05: Develop unit tests for document generation process (ID: 8005)

US-09: Performance Optimization and Error Handling

As a developer,
I want to ensure the tool performs efficiently and handles errors gracefully,
So that users have a smooth experience even with large projects or API issues.

Acceptance Criteria:

  • The tool processes large projects (up to 1000 files) within a reasonable time frame
  • Proper error handling is implemented for file operations, API calls, and document generation
  • Users receive clear and helpful error messages
  • The tool can recover from minor errors and continue processing

Tasks:

T-09-01: Implement asynchronous processing for API calls and file operations (ID: 9001)

T-09-02: Optimize file reading and writing operations (ID: 9002)

T-09-03: Develop comprehensive error handling strategy (ID: 9003)

T-09-04: Implement user-friendly error messages and logging (ID: 9004)

T-09-05: Conduct performance testing with large sample projects (ID: 9005)

Developer Notes
